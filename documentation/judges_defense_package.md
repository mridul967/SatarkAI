## 1. The Core Innovation: Hybrid Ensemble Inference

Unlike standard fraud detection systems that use simple "Rule Engines," SatarkAI uses a **Dual-Layered Neural Architecture** that combines tabular statistics with topological relationship mapping.

### Pillar A: Topological Graph Memory (GATv2)
- **Concept**: Fraudsters rarely act in isolation. They form "Money Mule" networks where money hops through 4-5 accounts to hide its origin.
- **Why it wins**: We use **Graph Attention Networks (GATv2)** (`backend/ml/train_gnn.py`). The "Attention" mechanism allows the AI to focus on specific suspicious *relationships* in a transaction network, catching money laundering that standard AI would miss entirely.

### Pillar B: High-Velocity Gradient Boosting (LightGBM)
- **Concept**: Standard graph models can be slow. We pair the Graph Engine with **LightGBM** (`backend/ml/train_lgbm.py`) for sub-10ms inference.
- **Why it wins**: This layer handles "Velocity Attacks" (e.g., a card being used in 3 different cities within 5 minutes).

## 2. Advanced Features for Judges

### 4-LLM Forensic Consensus Matrix
- **File**: `backend/services/llm_service.py`
- **Logic**: For every critical fraud, SatarkAI queries a matrix of 4 LLMs (Claude, GPT, Gemini, Groq). If the ML scores are high, the LLMs analyze the "Neural Narrative."
- **Proof**: If one model (e.g., GPT-4) hits a rate limit, the system automatically falls back to the others, ensuring the **FMR-1 RBI Report** is never empty.

### Immutable Audit Ledger (Blockchain)
- **File**: `backend/services/blockchain_service.py`
- **Proof**: Every high-confidence fraud signal is "Hashed" and committed to the **Polygon Blockchain**. This creates a "Digital Chain of Custody" that prevents bank insiders from deleting fraud records.

### Federated Learning with DP-SGD
- **File**: `backend/services/federated_service.py`
- **Concept**: Banks cannot share raw customer data due to privacy laws. 
- **Proof**: We use **Differential Privacy (DP-SGD)**. We only share "Gradients" (mathematical updates), meaning the model learns from *all* banks' data without ever seeing a single real customer's name.

If the judges ask about **Bias** or **Ethical AI**, tell them this:

1. **Synthetic Bias Mitigation**: We used the `attack_sampler.py` to balance our training data. This ensures the AI doesn't unfairly flag specific demographics or low-income transaction patterns.
